# PRD: 需求发布 App（Reverse 闲鱼）

本 PRD 摘要自 requirement.md，用于 Task Master parse-prd。

1. 产品定位：以“求购”为核心的信息撮合平台，支持同城优先与跨城（线下面交/快递）。
2. 用户与场景：有明确需求的买家、具备供给的卖家；典型场景包括二手/缺货/定制件。
3. 关键决策（已确认）：
   - 价格形态：支持单一期望价与价位区间（exact/range + price_min/price_max）。
   - 城市与跨城：默认同城优先，提供“一键只看同城”切换。
   - 实名强度：发布与报价强制实名，未实名仅可浏览。
4. MVP 范围（P0）：
   - 账号与实名登录（微信/支付宝其一）、发布需求、卖家报价、达成后线下面交与快递两种交易流程、关注订阅、站内通知、基础风控。
5. 关键流程：
   - 实名登录 → 发布需求 → 收到报价 → 沟通协商 → 线下或快递交易 → 评价/申诉。
6. 数据要点：
   - 求购（需求）：title, desc, category, city, is_local_only, price_type, price_exact, price_min, price_max, ship_pref
   - 报价：amount, shipping_fee, shipping_type, cod, lead_time, message
7. 风控与指标：
   - 违禁品、欺诈、频控；核心指标：发单转化、应答时效、成交率、争议率。

--
请据此生成任务，按模块拆分：认证、需求发布、检索/排序/同城优先、报价与交易（线下/快递）、通知、风控、数据指标、关注订阅、配置后台。
